⚔️ Batalha de Modelos & Engenharia de Prompt
📝 Descrição do Projeto

Este projeto consiste em um experimento prático de Engenharia de Prompt aplicado a Inteligência Artificial, no qual foi desenvolvido um prompt estruturado em XML com o objetivo de gerar automaticamente uma página HTML Single Page (SPA) com CSS e JavaScript integrados.

O experimento teve como foco avaliar como diferentes modelos de IA interpretam e executam um mesmo prompt estruturado, analisando aspectos como precisão do código, criatividade, quantidade de tokens utilizados e presença de erros de sintaxe.

A página solicitada no prompt representa um MVP (Minimum Viable Product) da plataforma ImobiFlow, um sistema SaaS de gestão imobiliária multi-tenant voltado para imobiliárias, corretores autônomos e administradores.

Durante a atividade, o mesmo prompt foi testado em diferentes modelos de IA para comparar seus resultados e desempenho.

📊 Funcionalidades Geradas pelo Prompt

O prompt estruturado solicitava que a IA criasse uma Single Page Application contendo diversos elementos de interface para um sistema imobiliário.

Principais requisitos implementados:

Menu de navegação funcional com âncoras
Dashboard com métricas do corretor
Sistema de gestão de leads em estilo Kanban
Cards de imóveis com status (Ativo, Reservado, Vendido)
Galeria de imóveis para visualização pública
Botão flutuante de WhatsApp
Interface responsiva com foco em Mobile-First

🧠 Resultados e Aprendizados

Principais conclusões do experimento:

Gemini e Claude demonstraram melhor compreensão da estrutura XML do prompt.
Houve diferença significativa na quantidade de tokens gerados entre os modelos.
Alguns modelos apresentaram maior precisão técnica na geração de código.

Com base na experiência:

Gemini se mostrou eficiente para prototipagem rápida.
Claude demonstrou bom desempenho em códigos mais estruturados e complexos.

📌 Conclusão

O experimento demonstrou que a qualidade do resultado gerado por IA depende fortemente da estrutura do prompt utilizado. Prompts bem definidos, organizados e detalhados aumentam significativamente a precisão das respostas.

Além disso, a comparação entre diferentes modelos evidenciou que cada IA possui pontos fortes distintos, sendo algumas mais eficientes para prototipagem rápida enquanto outras apresentam maior precisão técnica na geração de código.

# 📌 Sobre o Projeto

Este projeto apresenta um experimento de **Engenharia de Prompt**, no qual foi criado um **prompt estruturado em XML** com o objetivo de gerar automaticamente uma **Single Page Application (SPA)**.

O sistema simulado chama-se **ImobiFlow**, uma plataforma SaaS voltada para **gestão imobiliária**, com funcionalidades voltadas para:

- Corretores de imóveis
- Imobiliárias
- Administradores de sistema

O experimento foi utilizado para testar **como diferentes modelos de IA interpretam prompts estruturados**, analisando:

- Qualidade do código gerado
- Criatividade das soluções
- Quantidade de tokens utilizados
- Precisão na interpretação do XML

---

# 🧠 Objetivo da Experiência

O objetivo principal foi comparar o desempenho de diferentes **Modelos de Linguagem (LLMs)** ao receberem o mesmo prompt estruturado.

Foram avaliados critérios como:

✔ Qualidade da interface gerada  
✔ Estrutura do código HTML  
✔ Organização do CSS  
✔ Interatividade em JavaScript  
✔ Uso de tokens  
✔ Capacidade de seguir instruções complexas

---

# 🧪 Estrutura do Prompt

O prompt foi organizado utilizando **XML**, permitindo separar claramente cada parte da instrução.

Principais componentes:

<tarefa> <objetivo> <diretrizes\_design> <business\_logic\_requirements> <technical\_stack\_constraints> <output\_rules>

Essa estrutura permite que o modelo de IA **compreenda melhor cada parte da tarefa**, aumentando a qualidade do resultado gerado.

# 📈 Resultados Obtidos

A análise demonstrou que **nem todos os modelos interpretam prompts complexos da mesma forma**.

Observações importantes:

- Alguns modelos priorizam **criatividade visual**
- Outros focam mais na **estrutura correta do código**
- A quantidade de **tokens varia bastante entre modelos**

Isso evidencia a importância de escolher **a ferramenta adequada para cada tipo de tarefa**.
